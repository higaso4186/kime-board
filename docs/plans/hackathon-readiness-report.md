## 5. ハッカソン向け最短パス

**目標**: 会議メモを貼り付け → Agent が解析 → 決裁一覧に反映、までを 5 分でデモできるようにする。

| ステップ | 対応内容 |
|----------|----------|
| 1 | Web 会議フォームを `POST /api/projects/:id/meetings` に接続 |
| 2 | API + Agent をローカルで起動（DISABLE_CLOUD_TASKS の場合は Agent を直接叩く or 簡易キュー） |
| 3 | Firestore Emulator または本番 Firestore を使い、会議作成 → Agent 解析 → コールバック → 決裁作成 の流れを成立させる |
| 4 | Web を apiDataMode=production 相当にし、`/api/projects/:id/snapshot` で Firestore の実データを表示 |

※ Agent のルールベース解析は「決裁:」「選択肢:」等の行プレフィックスに対応。デモ用にサンプル議事録を用意すると伝わる。

## 5.1 プロダクトの立ち位置（非競合方針）

- 本プロダクトのアクションは、決裁を前に進めるための進行管理に限定する。
- 個別タスクの詳細管理（粒度の細かい担当・期限・工数・スプリント運用）は既存のタスク管理ツールを前提にする。
- つまり「タスク管理ツールを置き換える」のではなく、「意思決定の流れを可視化して前進させる」ための役割に集中する。
